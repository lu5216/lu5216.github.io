3.2.1 *查看修改的状态（status）
  • 作用：查看的修改的状态（暂存区、工作区）

  1. 绿色：暂存区
  2. 红色：未暂存


PS：文件输入了新内容，会导致该文件回退到未暂存状态

3.2.3 *提交暂存区到本地仓库（commit）
  1. git commit -m “xxx”

    PS：-m 是 –message的缩写
    PS: “xxx” 是注释内容，随便写，之后可以查看

  1. 再次查看状态


3.2.4 *查看提交日志（log）

:::warning
命令形式：git log [option] 或 git log
常用：git log –pretty=oneline –abbrev-commit –graph –all –decorate
:::
:::warning

  • option
    • –all 显示所有分支
    • –pretty=oneline 将提交的信息显示为一行
    • –abbrev-commit 使得输出的commitId更简短
    • –graph 以图的形式显示
      :::
      1. –pretty=oneline 将提交的信息显示为一行


  1. –abbrev-commit 使得输出的commitId更简短


  1. –all 显示所有分支


  1. –graph 以图的形式显示


  1. –decorate 以图的形式显示 （与4效果相同）


3.2.5 *版本回退 （reset）

  • 作用：版本切换
    :::warning

  • 命令形式：git reset –hard commitID （慎用）

    • commitID 可用使用 git-log 或 git log 指令查看

  • 查看已删除的提交过的记录：git reflog
    :::

4、分支

几乎所有的版本控制系统都以某种形式支持分支。
使用分支意味着你可以把你的工作重开发主线上分离开来进行重大的Bug修改、开发新的功能，以免影响开发主线。

4.1 查看与创建本地分支

4.1.1 查看本地分支

:::warning
命令：git branch
:::

4.1.2 创建本地分支

:::warning
命令：git branch 分支名
:::

4.2 *切换与合并分支

4.2.1 切换分支

:::warning
命令：git checkout 分支名
命令：git checkout-b 分支名 （切换到一个不存在的分支，创建并切换）
:::
HEAD -> 指向哪，哪里就是当前分支

4.2.2 合并分支

一个分支的·提交可以合并到另外一个分支
:::warning
命令：git merge 分支名
:::

4.3 删除分支与解决冲突

4.3.1 删除分支

不能删除当前分支，只能删除其他分支

:::warning
命令：git branch -d 分支名 （删除分支时，需要做各种检查）
命令：git branch -D 分支名 （不做任何检查，强制删除）
:::
